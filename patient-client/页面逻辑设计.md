# 智环引诊 - 患者客户端页面逻辑设计

## 一、整体架构设计

### 1.1 设计原则
- **用户体验优先**: 简化操作流程，减少点击次数
- **适老化设计**: 大字体、大按钮、清晰的视觉层级
- **无感化交互**: NFC自动感应、语音交互
- **关怀延伸**: 家属绑定、远程关怀

### 1.2 技术架构

## 四、页面逻辑设计

### 4.1 首页逻辑 (home)

**功能**: NFC手环感应入口

**状态**:
- `idle`: 空闲状态，等待感应
- `detecting`: 正在检测手环
- `detected`: 检测成功
- `error`: 检测失败

**流程**:
```
进入首页 → 自动检测NFC → 
├─ 检测成功 → 跳转到就诊概览页
└─ 检测失败 → 显示错误提示，提供手动绑定入口
```

**交互**:
- 点击"开始就诊" → 跳转到overview页
- 点击"手动绑定手环" → 跳转到nfc-bind页

---

### 4.2 就诊概览页逻辑 (overview)

**功能**: 患者就诊信息总览

**状态**:
- `active`: 当前就诊中
- `completed`: 就诊完成
- `pending`: 等待就诊

**流程**:
```
加载患者信息 → 加载就诊任务 → 显示进度 → 
├─ 点击"下一步任务" → 跳转到task-detail页
├─ 点击"查看报告" → 跳转到reports页
├─ 点击"复诊提醒" → 跳转到reminder页
├─ 点击"语音咨询" → 跳转到chat页
└─ 点击"紧急求助" → 跳转到emergency页
```

---

**功能**: 显示当前任务详情和导航指引

## 六、交互设计规范

### 6.1 触控反馈

| 交互类型 | 反馈方式 | 时间 |
|----------|----------|------|
| 点击按钮 | 缩放 + 阴影变化 | 200ms |
| 页面切换 | 平滑滚动 | 300ms |
| Toast提示 | 淡入淡出 | 3000ms |
| 加载状态 | 骨架屏 + 加载动画 | 直到完成 |

```
语音按钮 → 点击 → 显示"正在听您说话..." → 
├─ 3秒后 → 显示"语音识别完成" → 执行对应操作
└─ 点击取消 → 停止录音
```

### 6.3 NFC交互

```
进入首页 → 自动检测NFC → 
├─ 检测到 → 显示"手环识别成功" → 跳转到overview页
└─ 未检测到 → 显示NFC动画 → 等待用户操作
```

### 8.2 数据缓存

```javascript
const cache = {
    patient: null,
    tasks: null,
    reports: null,
    lastUpdate: null
};

function getCachedData(key) {
    const now = Date.now();
    const data = cache[key];
    // 5分钟内有效
    if (data && now - cache.lastUpdate < 5 * 60 * 1000) {
        return data;
    }
    return null;
}
```

## 十一、总结

### 11.1 设计亮点

1. **无感化入口**: NFC自动感应，减少操作步骤
2. **适老化设计**: 大字体、清晰导航
3. **关怀延伸**: 家属绑定，远程关怀
4. **智能交互**: 语音问答，便捷操作
5. **紧急求助**: 一键SOS，快速响应

### 11.2 核心价值

- **提升就诊效率**: 智能导航、实时队列
- **增强患者体验**: 语音交互、关怀模式
- **保障安全**: 紧急求助、位置追踪
- **数据透明**: 报告随时查看

### 11.3 实施建议

1. **第一阶段**: 核心就诊流程(首页、概览、导航、队列、报告)
2. **第二阶段**: 扩展功能(评价、轨迹、家属关怀)
3. **第三阶段**: 进阶功能(在线挂号、缴费、配送)
